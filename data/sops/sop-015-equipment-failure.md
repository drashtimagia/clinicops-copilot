# SOP-015 — Equipment Failure and Escalation

## Purpose
Standardize the clinic response when a medical device cannot be safely restored through basic troubleshooting.

## Trigger Conditions
Use this SOP when:
- a device fails repeatedly
- troubleshooting steps do not resolve the issue
- the device displays persistent fault codes
- there is suspected electrical, sensor, motor, or battery failure

## Immediate Actions
1. Stop using the device if function is unreliable.
2. Label device as out of service if needed.
3. Move workflow to alternate equipment when available.
4. Record the failure in the incident log.

## Basic Triage
Before escalation, confirm:
- power source
- cables/tubing/leads are connected properly
- consumables are seated correctly
- simple restart has been attempted once

Do not repeat the same failed step multiple times.

## Escalation Criteria
Escalate immediately for:
- overheating
- burning smell
- electrical fault
- repeated shutdown
- motor failure
- unverified temperature control
- unsafe infusion or monitoring accuracy

## Technician Routing
Primary escalation target:
- Biomedical Technician

Secondary escalation target:
- Operations Manager if workflow disruption is significant

## Communication Template
"Device failure confirmed. Unit removed from active use. Biomedical technician review required."

## Reroute Guidance
If alternate equipment is available:
- reroute patient/device workflow
- prioritize critical cases first
- communicate expected delays to affected staff

## Closure Criteria
Issue may be closed only when:
- device passes functional check
- technician clears device for use
- operational impact is updated
